Description and Guide

Description

Since modern mobile devices accompany their owners throughout the day and during all day-to-day activities, analyzing contextual/sensory data collected from them can be used for precise recognition of user's activity and context which is invaluable for proper decision making in Smart City content delivery services.

The POI recommendation is the main communication channel for engaging end users of smart city guide - like applications. The Context Aware Recommendation SE enables developers to recognize user's activity and put it into context. This knowledge is used to modify weights in recommendation matrices so that end user's activity and context is properly addressed through POI recommendation.

The Activity and Context Recognition module is used for classification and recognition of user's activity and context based on contextual and sensory data provided by mobile devices wearables. The Recommendation Matrix Preparation module uses knowledge of user's context and activity and POI attributes (collected from open POI repositories) to adapt weights of POI recommendation matrix. This way the POI recommendation is adapted to end user activity and context. Additionally, as part of the solution we also provide the mobile application module which collects contextual data from mobile devices (location, SSIDs of surrounding WiFi networks and sensory data) and WiFi APs (user's utilization of the WiFi internet access). As alternative, developers can use the Orion Publish/Subscribe Context Broker GE for obtaining contextual and sensory data from mobile devices and wearables.

Use the Android demo application for testing the main functionality of the SE through a simple smart city guide which recommends POIs (collected from the Foursquare API) based on your activity and context. By tapping the “Recommend” button at the bottom you will be recommended with 5 POIs of the relevant category based on your current activity and location. In the settings of the application you can specify base URL for the Activity and Context Recognition module and Recommendation Matrix Preparation module of the SE if you have deployed them yourself. Also, you can choose to pause the sensory data stream (please note that the sensory data stream start as soon as you run the application and will not stop until you exit the application or stop it manually in the settings).
